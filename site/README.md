# site

The public site for **cascade** — a type-driven typography spec and renderer. It showcases the CSS
renderer's output with a live specimen (a port of the original PoC viewer), and will grow into the
project's public pages. It is a workspace member here, alongside the library crates.

It is a **consumer** of cascade: it depends on `cascade-css` and renders the stylesheet **in-memory
via its API** (`Css::render(&Config)`, in `src/css.rs`, served at `/css/cascade.css`) — live, any
config, no vendored files. (cascade builds as a normal dependency because its spec types are
generated by `build.rs` -> `OUT_DIR`.)

## Stack

Mirrors [wine-app](../../wine-app): **Axum + Askama + datastar + Vite + PicoCSS**. It drops the
stateful half (no Postgres, no wasm/offline/PWA) — a stateless content + specimen site. Pico is
loaded to demonstrate cascade coexisting with another CSS library.

The chrome (header, Options dropdown, tabs, sun/moon toggle) is a faithful port of the PoC; it
drives the specimen — which loads in an iframe, isolated from Pico and styled by cascade — via the
specimen's `setCascade()` hook. The controls (scale/body/heading/notes) map to the class + `data-*`
attributes the renderer emits.

## Develop

From this directory (`cascade-typeset/site`), with `cargo`, `node`, `pnpm`:

    pnpm -C web install && pnpm -C web build   # vendor datastar + Vite build -> web/dist
    cargo run                                  # http://localhost:8080  (renders /css/cascade.css in-memory)

`BIND_ADDR` (default `0.0.0.0:8080`) sets the listen address. For a frontend dev loop, run
`pnpm -C web watch` alongside `cargo run`. (There is no task runner — the cascade library's own
operations are `cargo run -p cascade-cli -- <build|dist|measure>`; everything else is `cargo`/`pnpm`.)

## Layout

    src/            Axum server (page handlers, in-memory CSS, Vite manifest resolution)
    templates/      Askama templates: index.html (chrome) + sample.html (specimen)
    web/            Vite frontend (datastar + Pico + the control JS)
    web/public/     static assets served as-is (favicon, brand marks)
