# site -- conventions

## What this is

The public cascade site: a CONSUMER of the cascade typography library, a workspace member alongside
the library crates. It depends on `cascade-css` (a path dependency, `../cascade-css`) and renders the
stylesheet IN-MEMORY via its API -- `cascade_css::Css::render(&Config::default())` in `src/css.rs`,
served at `/css/cascade.css`. This is the library-as-API path: live, any config, no vendored files.
(cascade builds fine as a dependency because its spec is generated by `build.rs` -> `OUT_DIR`, not a
proc-macro that assumes it is the top-level project. cascade also ships a committed distribution at
`../dist/css` for non-Rust consumers; this site does not need it.)

## Stack

Mirrors wine-app: Axum + Askama + datastar + Vite + PicoCSS. Drops wine-app's stateful half (no
Postgres/sqlx, no wasm/offline/PWA) -- a stateless content + specimen site. Pico is loaded to prove
cascade coexists with another CSS library; the chrome reconciles Pico's defaults (margins, markers)
where they'd fight the ported layout.

- The chrome (a port of the PoC viewer) drives the specimen -- loaded in an iframe, isolated from
  Pico -- via its `setCascade()` hook. Controls: scale, body/heading typeface (`bundle-*` /
  `heading-*` classes), notes (`.banded` / `.centered`), and a light/dark toggle.

## Code style

- **No emojis in code or comments.** Not in `.rs`, `.toml`, `.ts`, Dockerfiles, config, or any
  comment. Emojis are permitted ONLY in content rendered to the end user (Askama templates / HTML
  shown in the browser). Keep code ASCII-clean. (Vendored third-party files such as
  `web/src/vendor/datastar.js` are not ours to modify.)

## Dev loop

    just build-web   # vendor datastar + Vite build -> web/dist
    just run         # cargo run -- http://localhost:8080 (CSS rendered in-memory)
